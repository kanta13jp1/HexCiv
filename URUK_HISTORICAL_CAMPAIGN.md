# ウルク史実キャンペーン — 設計確定事項と第1基盤

## 目的

既存のランダム4Xを残したまま、史実の位置・年代・成立条件を持つ別モードを追加する。
第1完成版は紀元前4000～3000年の南メソポタミアを扱う「ウルク—都市の夜明け」。
約50ターンの縦切り版で、都市化・河川農業・身分人口・実在物資・交易を段階的に遊べるようにする。

「歴史上のすべて」は有限の完了宣言ではなく、出典付き安定IDを後方追加する継続台帳として扱う。

## `/grill-me` で確定した製品判断

- 核はCivilization型の1人用4X。1ヘックス1部隊で世界地図上のターン制戦闘。
- 大量の歴史要素は、それぞれ固有効果を持つゲーム要素にする。
- ゲームバランスより史実再現を優先する。
- 文明・地域・最速登場年代を原則固定し、時代尚早の研究は不可。
- 世界地図上で国家が史実年代・史実地域に成立する。結果は固定せず、地理・人口移動・外交圧力を条件として再現する。
- プレイヤーは国家を操作し、国家滅亡で敗北。後発国家は建国直前の史実スナップショットから開始する。
- 長期キャンペーンは時代ごとに1ターンの年数を変え、全体を約300～400ターン・10～20時間に収める。
- 勝利は存続・軍事・科学・文化・経済。軍事等は即時勝利、存続は最終ターン判定。
- ウルク編の主軸は灌漑・人口・食料・都市周辺タイルの管理。灌漑はまず河川隣接農地の単純ボーナス。
- 人口は農民・職人・神官・兵士・奴隷などの身分別。奴隷制は強制、権利欠如、逃亡、反乱、社会不安、解放まで一体で扱う。
- 経済は大麦、羊毛、葦、粘土、木材、石材、銅など実在物資を在庫・消費・交易へ接続する。
- 学術的推定を使う場合は「確実・有力説・推定」を明示し、図鑑の各項目に出典・年代幅・異説を表示する。
- 画像は博物館等の利用可能な資料を優先。不足時の生成画像は「復元イメージ」と権利条件を明記する。
- 音楽は確認可能な楽器を参考にした現代制作であり、当時の旋律そのものとは表示しない。
- 固有名詞は日本語通用名・原語・学術転写・異綴りへ対応する。
- Windows PC専用。日本語表示のみだが、ローカライズキーを全データに持たせる。
- 初心者向けに短い導入チュートリアルと任意の顧問を用意し、人口配置・生産・交易を個別に自動化できる。
- UIは簡潔／詳細切替。一般的な内蔵GPUで1080p・30fps、通常のAI待ち時間5秒以内を目標。
- 美術は歴史復元を意識したローポリ3D。移動・建設・収穫・洪水・戦闘・都市成長など情報を伝えるアニメーションを優先。
- 歴史要素はJSON／CSV中心のデータ駆動方式とし、開発時生成・史実確認・権利確認を通過した素材だけを収録する。

## 第1マイルストーンの実装

- `Assets/Resources/Campaigns/uruk_4000.json`
  - schema version 1、安定ID、ローカライズキー、日本語文、史料確度、出典参照。
  - 32×20の固定南メソポタミア地図、推定した下流河道、湿地側へ広がる水域。
  - ウルク、エリドゥ、ウル、ラガシュ地域、ニップール、キシュ地域、スシアナ交易圏、南部湿地共同体の8勢力。
  - 大麦、エンマー小麦、羊・羊毛、葦、沖積粘土、木材、石材、銅、魚の9物資。
- `Core/HistoricalCampaign.cs`
  - JSONデータ契約、ローカライズ、出典、確度、年代、勢力、物資、河川の検証。
  - 50ターンを紀元前4000年から前3000年へ割り当てる年代変換。将来は複数の可変年代区間を持てる。
- `Core/HistoricalCampaignFactory.cs`
  - 手続きマップを使わず固定JSONから既存Core互換の8勢力GameStateを構築。
  - 個人名が不明な指導者を捏造せず、「氏名不詳の祭司王」等の役職名で表示。
- `Core/HistoricalCampaignSave.cs`
  - `mode=historical_campaign` とcampaign IDを持つ外側エンベロープで、既存ランダムゲームのセーブと区別。
  - 内側のGameStateは既存SaveLoadへ委譲し、既存の移行処理を再利用。

## 地理表現上の注意

固定マップは現代の河道・海岸線をそのまま描いたものではない。UNESCO資料は紀元前5000～3000年頃に海水面が現在より約200km内陸へ達し、ウルク・ウル・エリドゥが淡水湿地の縁辺に位置したことを説明している。一方、個々の河道・政治境界を紀元前4000年の一点へ確定することはできない。このため地図全体を `inferred`、主要遺跡の存在を `probable` または `confirmed` として分離した。
