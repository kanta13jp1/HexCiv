# 市場・交易・地域産業システム

最終更新: 2026-07-22 / 第1弾

## 目的

生活技術台帳の72件を閲覧項目だけで終わらせず、人口社会・国庫・補給・研究・文化へ接続する。ただし、現存する料理、踊り、歌、武術などを一律の交換可能な商品として扱わない。個別項目は担い手の知識を含む**地域産業・文化実践**として発展し、市場で取引する対象は次の5つの抽象財に限定する。

| 抽象財 | 主な生産源 | 主な需要 |
|---|---|---|
| 食料 | 農民、穀物庫、料理文化 | 全人口 |
| 素材 | 都市、採鉱、作業場、地域産品・武器産業 | 都市と軍事 |
| 製品 | 工人、作業場、名産品・名物 | 人口と建物 |
| 知識 | 学者、図書館、踊り・歌・武術の継承 | 都市と学者 |
| 輸送 | 都市、車輪、建築学、船・車・飛行機・ロケット | 都市、部隊、戦争 |

## 毎ターンの処理

1. 全文明をID順に並べ、市場方針と市場アクセスを確定する。
2. 人口階層、建物、技術、発展済み地域産業から5財を生産して在庫へ加える。
3. 国内需要を在庫から充足し、不足量と地域価格を算出する。
4. 平和な文明間で、輸出側の余剰・輸入側の不足・市場アクセス・輸送在庫・首都間距離から交易量を決める。
5. 交易価値を輸出国の収入、輸入国の支出へ加え、需要充足度と価格を更新する。

乱数は使用しない。同じ状態からは同じ交易結果、価格、収支、地域産業発展が得られる。

## 市場方針

| 方針 | 効果 |
|---|---|
| 自給優先 | 食料・素材+25%、市場アクセス-10 |
| 均衡市場 | 生産と交易を標準状態に保つ |
| 輸出振興 | 製品・輸送+25%、市場アクセス+10、輸出在庫を残しにくい |
| 戦時動員 | 素材・輸送+25%、知識-25%、対外交易能力低下 |

AIは戦時なら戦時動員、需要充足60%未満なら自給優先、国庫不足または輸出好調なら輸出振興、それ以外は均衡市場を選ぶ。

## 地域産業と生活技術72件

- 各文明は自身の6地域に属する12件を候補とする。
- 技術数の条件を満たし、最初は即時、その後は8ターン間隔で最大1件ずつ発展する。
- 名産品・名物は製品、料理は食料、船・車・航空機・ロケットは輸送へ寄与する。
- 踊り・歌・武術は「販売物」ではなく、教育・実践・継承による知識と文化への寄与として扱う。
- 歌詞、楽譜、設計図、技法の詳細は複製しない。

## 他システムへの接続

- 需要充足は都市満足度へ-6～+4の範囲で影響する。
- 需要充足と製品在庫は都市生産を85～115%の範囲で補正する。
- 知識在庫と交通技術産業は科学へ、料理・踊り・歌は文化へ小さく寄与する。
- 十分な輸送在庫は補給距離を+1する。
- 交易黒字は国庫収入、交易赤字は支出として国家運営へ表示する。
- 交戦中の2文明は直接交易できない。

## UI・セーブ

- 右側の「市場」またはF4で開く。
- 5財の在庫、生産、需要、価格、需給、市場アクセス、需要充足、輸出入、交易収支、主要相手、地域産業を表示する。
- 木箱と双方向矢印のアイコンを`Texture2D.SetPixels32`で実行時生成し、0.18秒のスライドフェードと既存SEを使う。
- SaveLoad version 14。version 13以前は均衡市場、各在庫3、市場アクセス50、需要充足75、地域産業なしで移行する。
